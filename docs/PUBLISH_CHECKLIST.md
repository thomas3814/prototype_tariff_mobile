# 공개 전 점검 체크리스트

이 체크리스트는 저장소를 다른 사람에게 공유하기 전에 빠르게 점검하기 위한 문서입니다.

## 1. 저장소 첫인상

- [ ] 저장소 이름이 이해하기 쉬운가
- [ ] 저장소 Description이 비어 있지 않은가
- [ ] README 상단에 프로젝트 목적이 2~3줄로 정리되어 있는가
- [ ] 현재 기준 최신 안정 버전이 무엇인지 명확한가 (`v15` 등)

## 2. 가져가는 방법 안내

- [ ] `Fork`, `Clone`, `Download ZIP` 차이가 README에 정리되어 있는가
- [ ] 사용 목적별 추천 방식이 적혀 있는가
- [ ] v15 태그 또는 Release 기준 안내가 있는가
- [ ] 기본 브랜치명(`main`/`master`)이 문서와 일치하는가

## 3. 실행 및 검토 편의성

- [ ] 실행 명령이 실제 프로젝트 기준으로 적혀 있는가
- [ ] 환경 변수 또는 설정 파일 위치가 안내되어 있는가
- [ ] 샘플 데이터/샘플 엑셀/템플릿 파일 위치가 정리되어 있는가
- [ ] 대표 화면 스크린샷이 최소 2장 이상 있는가

## 4. 외부 공유 안정성

- [ ] `v15` 태그가 생성되어 있는가
- [ ] Release 초안 또는 Release 본문이 준비되어 있는가
- [ ] 민감 정보가 포함된 파일이 제거되었는가
- [ ] `.env`, API 키, 내부 URL 등이 노출되지 않았는가
- [ ] 공개 가능한 라이선스가 결정되었는가

## 5. 협업 준비도

- [ ] `CONTRIBUTING.md`가 있는가
- [ ] `CHANGELOG.md`가 최신 상태인가
- [ ] 문서와 실제 코드 상태가 일치하는가
- [ ] 변경사항이 있으면 스크린샷/설명도 같이 반영했는가

## 6. 최종 권장 액션

공유 직전에는 아래 순서로 마무리하는 것을 권장합니다.

1. README 최종 검토
2. 대표 스크린샷 반영
3. `v15` 태그 생성
4. GitHub Release 발행
5. ZIP 다운로드 테스트
6. 다른 사람 기준으로 README만 읽고 시작 가능한지 확인

## 7. 테스트 메모

아래 항목을 직접 체크해보면 실사용성이 높아집니다.

- [ ] ZIP 다운로드 후 압축 해제만으로 구조를 이해할 수 있는가
- [ ] clone 후 바로 어떤 명령을 실행해야 하는지 보이는가
- [ ] fork 후 upstream 연결 가이드가 충분한가
- [ ] README만 읽고도 프로젝트 목적을 설명할 수 있는가
